# LexiFlow Documentation Pack

This folder contains the documentation package for explaining LexiFlow to a
technical audience.

## Documents

| Document | Purpose |
|---|---|
| `system_architecture.md` | Overall system architecture and high-level data flow |
| `project_documentation.md` | Detailed project explanation, capabilities, schema, demo flow |
| `component_integration.md` | Component and logical agent interaction diagrams |
| `rag_and_mcp_usage.md` | Separate explanation of how RAG and MCP are used in this project |
| `technical_audience_ppt.md` | Slide-by-slide technical presentation draft |

## Recommended Reading Order

1. `project_documentation.md`
2. `system_architecture.md`
3. `component_integration.md`
4. `rag_and_mcp_usage.md`
5. `technical_audience_ppt.md`

## Diagram Format

The architecture diagrams use Mermaid syntax. They render automatically in many
Markdown viewers, GitHub, and documentation systems that support Mermaid.

## Presentation Format

`technical_audience_ppt.md` is a Markdown slide draft. It can be converted into
a `.pptx` file later using a slide tool such as Marp, Pandoc, or a custom
PowerPoint generation script.
